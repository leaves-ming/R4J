# RAG 项目代理工作指南

## 目标

本仓库用于构建一个基于 LangChain4j 的模块化单体 Java RAG 系统。
系统目标聚焦四项核心能力：

1. 文档摄取
2. 混合检索：Dense + Sparse + RRF
3. 带引用的答案生成
4. 基于 golden set 的效果评估

## 语言要求

除以下情况外，代理必须优先使用中文：

1. 代码本身的关键字、库名、类名、接口名、包名、配置前缀
2. 用户明确要求保留英文的内容
3. 行业内已经固定、翻译后反而更易歧义的少量技术术语

默认要求如下：

1. 对用户的回复使用中文
2. 代码注释使用中文
3. 文档说明使用中文
4. 提交说明、变更说明、评审意见优先使用中文
5. 新增文档时，只要能用中文表达，就不要写英文

如果必须保留英文术语，应尽量在上下文中给出中文语义，避免整段英文说明。

## 实现指导
修改已有代码时，遵循以下原则：

1. 从受影响范围最小的用例入口开始定位
2. 先判断影响的是摄取、查询、答案结构还是评估
3. 检查是否出现跨层越界依赖

## 配置规则

必须使用强类型配置对象。
不要在代码里到处散落字符串配置键，也不要大量使用 `@Value` 直接取值。

## 测试要求

优先编写验证契约与架构边界的测试，不要只覆盖理想路径。
如果代码和构建文件已经存在，代理应先跑最小相关测试，再视情况扩大验证范围。
如果仓库还没有构建工具或测试命令，必须明确说明“未能验证”，不能假装已经验证通过。

## 在本仓库中工作的要求
当前仓库以文档为主

## 完成任何变更前的检查清单
代理在结束工作前至少确认以下事项：

1. 变更是否遵守模块边界
2. 如果修改了答案生成，引用和可追溯性是否仍然完整
3. 新增配置是否为强类型并具备校验
4. 行为变化是否补充或更新了测试
5. 最终回复是否明确说明“验证了什么”和“没验证什么”
